# Code of Conduct

Jingle is a public engineering project for a local desktop agent. The community
works best when discussion is direct, specific, and respectful.

## Expected Behavior

- Be kind and clear.
- Assume contributors are trying to improve the project.
- Critique code, architecture, tests, and tradeoffs rather than people.
- Share enough context for others to reproduce bugs or review decisions.
- Respect maintainers' decisions about scope, safety, and project direction.

## Unacceptable Behavior

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Repeatedly derailing issues or pull requests after a maintainer has asked to
  stop.
- Using project spaces to distribute malware, exploit instructions, spam, or
  unrelated promotion.

## Reporting

Report conduct concerns to the maintainers through a private channel when one is
available. If there is no private channel yet, open a minimal public issue asking
for a maintainer contact without sharing sensitive details.

Maintainers may remove comments, close issues, block accounts, or take other
reasonable action to keep the project usable and safe.
