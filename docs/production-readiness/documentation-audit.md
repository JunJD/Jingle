- `archive`: useful historical context, but should not be indexed as current release guidance.

## Archive/Delete Policy

No deletion is recommended in the first execution wave unless a file is clearly duplicated by a new production doc and has no historical value. The safer first wave is:

1. Rewrite stale root release docs and docs index.
2. Add an `Archive / Research` section so historical docs stop being presented as current contracts.
3. Move only obviously historical docs in small batches if the diff stays easy to review.
4. Keep launch/blog assets separate from help/dev docs.
